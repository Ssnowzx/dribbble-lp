# Project Brief: dribbble

## 1. Executive Summary
**Product Concept:** Landing Page de altíssima conversão focada no nicho Web3/SaaS, desenvolvida para ser comercializada como um UI Kit/Template premium no Dribbble.
**Problem:** Falta de templates premium que unam design "eye-candy" impecável com arquitetura de código limpa e escalável para o ecossistema Web3.
**Target Market:** Designers, Desenvolvedores e Empreendedores no Dribbble/Gumroad/UI8.
**Value Proposition:** Estética visual impressionante (Bento Grid, Glassmorphism, Dark Mode) aliada a uma experiência de desenvolvedor superior.

## 2. Problem Statement
**Current State:** Muitos templates focam apenas no visual ou apenas no código, falhando em entregar o equilíbrio necessário para o público exigente do Dribbble.
**Impact:** Baixa conversão em vendas de templates e dificuldade de customização para o comprador final.
**Urgency:** O mercado Web3/SaaS demanda interfaces modernas e rápidas; este projeto preenche essa lacuna com o padrão "Dribbble-ready".

## 3. Proposed Solution
**Approach:** Landing Page imersiva utilizando as últimas tendências de design (Dark & Neon, Micro-interações) e tecnologias modernas (React, Framer Motion, Vanilla CSS).
**Differentiators:** Foco obsessivo em micro-detalhes (bordas de 0.6px, glows sutis, tipografia General Sans) e uma narrativa fluida de conversão.
**Vision:** Tornar-se um benchmark de template premium no Dribbble.

## 4. Target Users
### Primary User Segment: Template Buyers
- **Profile:** Freelancers e agências Web3.
- **Goals:** Lançar Landing Pages rápidas e visualmente impactantes para seus clientes.
- **Needs:** Código fácil de ler, design moderno e facilidade de deploy.

## 5. Goals & Success Metrics
### Business Objectives
- Criar um produto "Dribbble-worthy" com alto potencial de saves e shares.
- Gerar tráfego qualificado para as plataformas de venda (Gumroad/UI8).
### User Success Metrics
- Facilidade de customização (medida por feedbacks).
- Performance Lighthouse > 90.

## 6. MVP Scope
### Core Features (Must Have)
- **Hero Section:** Vídeo imersivo, Navbar glassmorphism e CTA detalhado.
- **Logo Cloud:** Marquee infinito de parceiros Web3.
- **Core Features (Bento Grid):** Grade assimétrica com glows radiais.
- **Developer Experience:** Bloco de código estilizado (macOS style).
- **Bottom CTA & Footer:** Fechamento forte e responsivo.
### Out of Scope for MVP
- Dashboard administrativo.
- Integração com backend complexo.
- Blog engine.

## 7. Technical Considerations
- **Frontend:** React (TypeScript) com Vanilla CSS (preferência do framework AIOX).
- **Animations:** Framer Motion ou CSS puro para micro-interações.
- **Typography:** General Sans (Fontshare) - Controle rigoroso de line-height.
- **Color Palette:** Pure Black (#000000), White with varying opacities.
- **UI Elements:** Bordas de 0.6px, glassmorphism, glows sutis.

## 8. Constraints & Assumptions
- **Constraint:** Prazo para postagem no Dribbble.
- **Assumption:** O comprador final possui conhecimentos básicos de React para customização.
- **Technical Constraint:** Manter o bundle size baixo apesar das animações.

## 9. Risks & Open Questions
- **Risk:** Performance prejudicada pelo vídeo em loop na Hero (Mitigação: compressão agressiva e lazy loading).
- **Open Question:** Qual plataforma de vendas será o foco principal inicial?
